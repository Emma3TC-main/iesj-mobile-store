---

# 📌 Resumen del Proyecto

**iESJ Mobile Store** implementa una solución mobile-first orientada a la venta de componentes tecnológicos y hardware computacional.

El proyecto fue desarrollado utilizando buenas prácticas modernas de ingeniería de software:

- Arquitectura desacoplada
- Componentización reutilizable
- Context API
- Hooks personalizados
- Persistencia local
- Navegación modular
- Sistema centralizado de estilos
- CI/CD básico con GitHub Actions
- Cloud Builds con Expo EAS

# 🏗️ Arquitectura del Proyecto

El proyecto sigue una arquitectura modular organizada por responsabilidades:

```txt
src/
├── api/
├── components/
├── constants/
├── context/
├── hooks/
├── navigation/
├── screens/
├── services/
├── styles/
└── utils/
````

## Características de Arquitectura

* Componentes reutilizables
* Separación de responsabilidades
* Context API para estado global
* Hooks personalizados
* Sistema centralizado de temas y estilos
* Navegación Stack + Bottom Tabs
* Escalabilidad y mantenibilidad

# ▶️ Scripts Disponibles

## Iniciar proyecto

```bash
npm start
```

## Ejecutar Android

```bash
npm run android
```

## Ejecutar iOS

```bash
npm run ios
```

## Ejecutar versión web

```bash
npm run web
```

## Verificar entorno Expo

```bash
npm run doctor
```

## Build Android Preview

```bash
npm run build:android
```

## Build Android Producción

```bash
npm run build:production
```

---

# ⚙️ Expo EAS Build

El proyecto utiliza Expo Application Services (EAS) para compilaciones cloud y despliegue móvil.

## Configuración inicial

```bash
eas build:configure
```

## Build Preview

```bash
eas build -p android --profile preview
```

## Build Producción

```bash
eas build -p android --profile production
```

# 🔄 Integración Continua (CI/CD)

El proyecto incorpora GitHub Actions para automatizar validaciones y verificaciones básicas del entorno Expo.

## Pipeline

```txt
Push
   ↓
GitHub Actions
   ↓
Install Dependencies
   ↓
Expo Doctor
   ↓
Expo Validation
   ↓
Build Verification
```

## Validaciones automáticas

* Instalación de dependencias
* Expo Doctor
* Validación de compilación
* Verificación de compatibilidad Expo SDK
